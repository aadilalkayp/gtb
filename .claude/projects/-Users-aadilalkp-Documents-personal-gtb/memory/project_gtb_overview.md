GTB OS is an internal + client-facing web platform for managing wedding transformation programs (Groom To Be for grooms, Glow To Be for brides). Services include skincare, fitness, and styling.

**Why:** The team currently manages everything via WhatsApp, spreadsheets, and phone calls. GTB OS centralizes the entire client lifecycle.

**How to apply:**

- Backend is shared for GTB and Glow To Be. Frontend branding differs by client type (groom/bride).
- CROs handle both sales and post-sale relationship management (no separate sales role).
- Consultants are contractors, tracked via the expense module for payments.
- Plans are admin-configurable with wedding-date-based scheduling rules.
- MVP includes a client portal (registration, payment proof upload, session ratings, view schedule).
- Key stakeholders: Ishak K (founder/client), Aadil (developer).
- SRS v2.0 is at `gtb_os_srs_v2.md`. Original rough doc is `gtp_rq.md`.
